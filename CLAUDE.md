# 이 작업 폴더에 대해

개발 연습용 폴더입니다.
현재 **호박마켓** (중고거래 웹사이트)을 단계적으로 만들어가고 있어요.

## 기술 스택

- **Next.js** — 웹페이지를 만드는 프레임워크 (React 기반)
- **Supabase** — 데이터베이스 + 로그인 기능을 제공하는 서비스
- **Vercel** — 만든 웹사이트를 인터넷에 올리는(배포) 서비스
- **Tailwind CSS** — 디자인을 빠르게 적용하는 도구

## 프로젝트 구조

```
ggm/
├── index.html          ← 럭키비키 가계부 (기존 프로젝트)
└── hobak-market/       ← 호박마켓 Next.js 프로젝트
```

## Claude에게 당부하는 것들

### 말하는 방식
- 개발 용어는 **비개발자도 이해할 수 있게** 쉽게 설명해줘
  - 예) "RLS를 활성화했습니다" → "다른 사람이 내 데이터를 못 보도록 보안 설정을 켰어요"
  - 예) "미들웨어를 작성했습니다" → "로그인 여부를 확인하는 문지기 코드를 만들었어요"
- **지금 뭘 하고 있는지** 자주 알려줘. 조용히 작업하지 말고 중간중간 현황 공유
- 완료 후에는 **무엇이 달라졌는지** 한눈에 보이게 정리해줘

### 작업 방식
- 사용자가 **지시한 것만** 단계적으로 수행해. 요청하지 않은 기능은 미리 만들지 마
- **자동으로 할 수 있는 건 최대한 직접 해**. 코드 작성, 데이터베이스 테이블 생성 등
- **사람이 직접 해야 하는 것** (Supabase 대시보드 설정, Vercel 배포 버튼 클릭 등)은
  빠뜨리지 말고 **쉬운 말로 단계별로** 안내해줘
- 개발이 끝나면 **항상 바로 확인할 수 있게** 브라우저에서 열어줘 (dev 서버 실행)

### 서버 관련
- 개발 서버는 `hobak-market/` 안에서 `npm run dev`로 실행
- 기존에 서버가 켜져 있으면 죽이고 새로 시작해줘
- 개발 서버 주소: **http://localhost:3000**

## 사람이 직접 해야 하는 설정 (한 번만)

아래 항목들은 Claude가 대신할 수 없어서 직접 해줘야 해요.

### Supabase 이메일 인증 설정
1. [Supabase 대시보드](https://supabase.com/dashboard) 접속
2. 프로젝트 선택 → 왼쪽 메뉴 **Authentication** → **URL Configuration**
3. **Site URL**: `http://localhost:3000`
4. **Redirect URLs**에 추가: `http://localhost:3000/auth/callback`
5. 나중에 Vercel 배포 후엔 실제 주소도 여기에 추가

### Vercel 배포 설정 (배포할 때)
1. [vercel.com](https://vercel.com) 접속 → GitHub 연동
2. `hobak-market` 폴더를 프로젝트로 선택
3. 환경변수 2개 입력:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
